# 实验报告元技能

## 适用场景

用户提供实验相关的任务（课程实验、科研实验），需要完成数据处理和报告生成。

## 思维框架

以下是思考顺序，不是固定步骤——根据实际情况跳过或调整：

### 1. 理解实验目的

- 阅读实验 PPT/文档（用 AutoOffice + VLM）
- 明确实验要验证什么、需要什么数据、预期产出
- 如果有不理解的内容（如文件名含义），主动向用户澄清

### 2. 整理实验资料

- 定位数据文件、采集记录、参数表
- 用 VLM 或 AutoOffice 读取非结构化资料（图片、扫描件）
- 复制数据到工作目录，建立清晰的文件组织

### 3. 阅读示例代码

- 如果提供了示例代码或参考实现，先阅读理解
- 确认工具链（MATLAB/Python/R）和可视化风格
- 保持与示例一致的工具链，避免产出与预期不符

### 4. 数据处理与代码编写

- 使用与实验匹配的工具链（搜索 CLI 方案优先）
- 生成中间结果（图表、数据表）
- 用 VLM 评估图表质量，进入优化循环

### 5. 报告生成

- 使用 AutoOffice 按模板生成报告
- 注意格式细节：LaTeX 公式在 docx 中的正确写法、长公式独占一行
- 确保图表正确嵌入、编号连续

### 6. 质量验证

- VLM 逐页审查报告
- 对照实验要求检查完整性
- 进入优化循环直至满足要求

### 7. 交付

- 导出最终文件（docx + PDF）
- 向用户汇报完成情况和关键发现

## 工具寻路

| 能力 | 优先工具 | 备选 |
|------|----------|------|
| 阅读 PPT/文档 | AutoOffice (officecli) | VLM 截图分析 |
| 数据处理 | 实验指定工具链 (MATLAB CLI / Python) | 搜索 CLI 替代方案 |
| 图表评估 | PolarPrivate VLM | screenshot_and_analyze |
| 报告生成 | AutoOffice (officecli generate) | 手动 Markdown 转换 |
| 格式校验 | AutoOffice (officecli validate) | VLM 视觉审查 |
